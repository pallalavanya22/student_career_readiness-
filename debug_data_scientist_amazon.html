📋 Mock Interview for data scientist at amazon

Technologies: python,java | Languages: python

🔧 1. Technical / Conceptual Questions

Q1: Describe how you would handle missing data in a dataset.
Sample Answer: Options: remove rows/columns (if few), impute with mean/median/mode, use KNN imputation, predictive imputation, or create a missing indicator feature. Choice depends on missingness pattern (MCAR, MAR, MNAR).
Q2: What is feature engineering and why is it important?
Sample Answer: Feature engineering creates new features from raw data to improve model performance. Examples: log transforms, interaction terms, binning, encoding categoricals, date decomposition. Often more impactful than model selection.
Q3: How would you approach an A/B testing experiment?
Sample Answer: Define hypothesis, choose metric, calculate sample size, randomly split users, run experiment for sufficient duration, check statistical significance (p-value < 0.05), and consider practical significance.
Q4: Explain the Central Limit Theorem and its practical applications.
Sample Answer: CLT states that sample means approach a normal distribution as sample size increases, regardless of population distribution. This enables hypothesis testing, confidence intervals, and A/B testing.
Q5: What are the assumptions of linear regression?
Sample Answer: Linearity, independence of errors, homoscedasticity (constant variance), normal distribution of residuals, no multicollinearity. Violations require transformations or different models.

💻 2. Coding Questions

Q1: Implement a decorator that caches function results (memoization).
Solution Approach: Use a dictionary to store results. The decorator wraps the function, checks if args are cached, returns cached result or computes and stores it. Python's @functools.lru_cache does this built-in.
Q2: Write a generator function that yields Fibonacci numbers.
Solution Approach: Use two variables a, b = 0, 1. In a loop: yield a, then update a, b = b, a+b. Generators are memory efficient — they produce values on-demand instead of storing all in memory.
Q3: Implement a function to find all permutations of a string.
Solution Approach: Use recursion: base case (single char returns itself). For each character, fix it and recursively permute the rest. Time: O(n!). Can also use itertools.permutations().

🤝 3. HR / Behavioral Questions

Q1: Describe a time when you had to learn a new technology quickly.
Sample Answer: When my team adopted a new framework, I dedicated evenings to documentation and tutorials, built a small prototype, and shared my learnings with the team within a week. I believe in learning by doing and documenting as I go.
Q2: How do you handle tight deadlines and pressure?
Sample Answer: I prioritize tasks by impact, break work into manageable chunks, communicate proactively with stakeholders about progress, and focus on delivering the MVP first. I also maintain a healthy work routine to stay productive.
Q3: What is your biggest weakness and how are you working on it?
Sample Answer: I sometimes spend too much time optimizing code before shipping. I'm learning to balance perfectionism with pragmatism by setting time-boxes for optimization and focusing on delivering working features first, then iterating.
Q4: How do you stay updated with the latest trends in data scientist?
Sample Answer: I follow industry blogs, subscribe to newsletters (like TLDR, Morning Brew Tech), attend webinars, participate in open-source projects, and regularly practice on coding platforms. I also follow key people in the data scientist space on Twitter/LinkedIn.

📈 4. Feedback & Improvement Tips

  1. Use the STAR method (Situation, Task, Action, Result) for all behavioral questions.
  2. Research amazon's recent products, blog posts, and engineering culture before the interview.
  3. Build a portfolio project using python,java and deploy it — this impresses interviewers at amazon.
  4. Practice at least 2-3 data scientist-specific coding problems daily on LeetCode or HackerRank.
  5. Practice explaining data scientist concepts to a non-technical person — this demonstrates deep understanding.
